# MAC16 -- 16-bit 乘累加器芯片设计

> "华大九天"企业赛题：基于全流程国产 EDA 工具的 MAC16 芯片设计

## 1. 项目简介

本项目实现了一个 16-bit 有符号乘累加器 (Multiply-Accumulate, MAC16)。MAC 运算是数字信号处理 (DSP) 和神经网络硬件加速中最基本且关键的运算单元，其核心运算为：

```
acc = acc + din_A × din_B
```

输入为 16-bit 有符号数（补码），累加输出为 32-bit 有符号数。采用华大九天全流程国产数字 EDA 工具完成 RTL 设计、仿真验证、Lint 检查、逻辑综合及网表导出。

## 2. 架构设计

### 2.1 整体架构

```
                  +-----------------+
  din_A[15:0] --->|                 |
  din_B[15:0] --->|   Multiplier    |---> product[31:0]
                  | (16x16 signed)  |         |
                  +-----------------+         v
                                        +-----------+
                  +-----------------+   | product   |
  acc_value  ---->|                 |<--+-----------+
  (feedback)      |     Adder       |
                  | (32+32 signed)  |---> sum[31:0], overflow
                  +-----------------+         |
                                              v
                  +-----------------+
  clk ---------->|                 |
  rst_n -------->|   Accumulator   |---> acc_out (feedback)
  en ----------->| (seq register)  |---> mac_out[31:0]
                  |                 |---> carry
                  |                 |---> out_ready
                  +-----------------+
```

### 2.2 模块划分

| 模块 | 文件 | 类型 | 功能 |
|------|------|------|------|
| `mac16` | `rtl/mac16.v` | 顶层模块 | 例化并连接各子模块 |
| `multiplier` | `rtl/multiplier.v` | 组合逻辑 | 16x16 有符号乘法，输出 32-bit 乘积 |
| `adder` | `rtl/adder.v` | 组合逻辑 | 32-bit 有符号加法，含溢出检测 |
| `accumulator` | `rtl/accumulator.v` | 时序逻辑 | 累加寄存器 + 输出寄存器，控制 en 行为 |

## 3. 接口说明

### 3.1 输入信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| `clk` | 1 | Input | 时钟信号 |
| `rst_n` | 1 | Input | 异步低有效复位，复位时所有寄存器清零 |
| `en` | 1 | Input | 使能信号（详见下方行为说明） |
| `din_A` | 16 | Input | 16-bit 有符号输入操作数 A（被乘数） |
| `din_B` | 16 | Input | 16-bit 有符号输入操作数 B（乘数） |

### 3.2 输出信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| `mac_out` | 32 | Output | 32-bit 累加结果输出（out_ready 有效时可读取） |
| `carry` | 1 | Output | 溢出标志，累加过程中结果超出 32-bit 有符号范围时为高 |
| `out_ready` | 1 | Output | 输出就绪信号，计算结束时为高电平 |

### 3.3 使能信号 (en) 行为说明

| 状态 | 行为 |
|------|------|
| `en=1` (高电平) | 乘法器和累加器工作。每个时钟周期读入 din_A、din_B 并进行乘累加运算。`out_ready=0`。 |
| `en` 1->0 (下降沿) | 计算结束。累加器将当前累加值锁存到 `mac_out` 输出寄存器，内部累加器清零，`out_ready` 拉高。 |
| `en=0` (低电平保持) | `mac_out` 保持上一次计算结果，`out_ready` 保持高电平。 |
| `en` 0->1 (上升沿) | 累加器从零重新开始新一轮累加计算，`out_ready` 拉低，`carry` 清零。 |

## 4. 设计细节

### 4.1 乘法器 (multiplier)

- 利用 Verilog `signed` 类型声明，直接进行有符号 16x16 乘法
- 输出 32-bit 有符号乘积，精确无截断

### 4.2 加法器 (adder)

- 将乘积与当前累加值进行 32-bit 有符号加法
- 溢出检测：两个同号数相加，结果符号位发生翻转则判定溢出

### 4.3 累加寄存器 (accumulator)

- 双寄存器架构：内部 `acc_out`（累加器）+ 输出 `mac_out`（结果保持）
- 通过 `en_d`（延迟一拍的 en）实现上升沿/下降沿检测
- `carry` 为粘性 (sticky) 标志：累加过程中一旦发生溢出即置位，直到新一轮计算开始才清零
- 异步低有效复位：`rst_n` 为低时立即清零所有寄存器

## 6. 仿真验证

### 6.1 赛题指定测试用例

| 步骤 | din_A | din_B | 期望累加值 | 说明 |
|------|-------|-------|-----------|------|
| 1 | 3 | 5 | 15 | 0 + 3x5 = 15 |
| 2 | -3 | -5 | 30 | 15 + (-3)x(-5) = 15+15 = 30 |
| 3 | 3 | -5 | 15 | 30 + 3x(-5) = 30-15 = 15 |

en 拉低后：`mac_out=15`, `out_ready=1`

### 6.2 扩展测试用例

Testbench (`tb/tb_mac16.v`) 共包含 14 组测试：

| 编号 | 测试内容 | 说明 |
|------|---------|------|
| 1 | 复位功能 | 验证 rst_n 清零所有输出 |
| 2 | 赛题指定三步测试 | 3x5, (-3)x(-5), 3x(-5) 连续累加 |
| 3 | 使能重启 | en 再次拉高后从零开始累加 |
| 4 | 有符号负x正 + 正转负 | (-100)x50, 然后 200x50 |
| 5 | 零值输入 | 0 x 12345 = 0 |
| 6 | 最大正值 | 32767 x 32767 |
| 7 | 最小负值 | (-32768) x (-32768) |
| 8 | 极值混合 | (-32768) x 32767 |
| 9 | 多周期有符号混合累加 | (-2)x3 + 4x(-5) + 6x7 = 16 |
| 10 | 平方和累加 | 1^2+2^2+3^2+4^2+5^2 = 55 |
| 11 | 溢出检测 | 连续累加触发 carry |

| 13 | 恒等乘法 | 12345 x 1 = 12345 |
| 14 | 多轮 en 开关 | 验证每轮累加独立从零开始 |

### 6.3 使用 Icarus Verilog 仿真

```bash
# 编译
iverilog -o mac16_sim rtl/multiplier.v rtl/adder.v rtl/accumulator.v rtl/mac16.v tb/tb_mac16.v

# 运行仿真
vvp mac16_sim

# 查看波形 (可选)
gtkwave mac16_wave.vcd
```

### 6.4 使用华大九天 ALPS/ALPS-GT 仿真

```bash
# 根据华大九天工具链文档执行
alps -f rtl/multiplier.v rtl/adder.v rtl/accumulator.v rtl/mac16.v tb/tb_mac16.v
```

### 6.5 Python 行为模型验证

```bash
python3 tb/verify_mac16.py
```

该脚本以 Python 实现了与 RTL 完全一致的行为模型，用于独立交叉验证算法正确性。

## 7. 时序约束 (SDC)

约束文件 `constraints/mac16.sdc` 定义了以下关键约束：

| 约束项 | 值 | 说明 |
|--------|-----|------|
| 时钟周期 | 10.0 ns | 对应 100MHz（可按需调整） |
| 时钟不确定度 | 0.3 ns | 包含 jitter 和 skew |
| 输入延迟 (max) | 3.0 ns | 数据信号，约 30% 时钟周期 |
| 输出延迟 (max) | 3.0 ns | 约 30% 时钟周期 |
| 输出负载 | 0.05 pF | |
| 最大扇出 | 20 | |
| 最大转换时间 | 0.3 ns | |
| 异步复位 | false path | rst_n 不参与时序分析 |
