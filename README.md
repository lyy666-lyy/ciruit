# MAC16 -- 16-bit 乘累加器芯片设计

> "华大九天"企业赛题：基于全流程国产 EDA 工具的 MAC16 芯片设计

## 1. 项目简介

本项目实现了一个 16-bit 乘累加器 (Multiply-Accumulate, MAC16)，支持有符号（补码）和无符号两种运算模式。MAC 运算是数字信号处理 (DSP) 和人工智能 (AI) 加速器中最基本且关键的运算单元，其核心运算为：

```
out = out + A × B
```

设计目标工作频率不低于 500MHz，采用华大九天全流程 EDA 工具完成从 RTL 设计到后端版图的全过程。

## 2. 架构设计

### 2.1 整体架构

```
                +------------------+
  a[15:0] ----->|                  |
  b[15:0] ----->|   Multiplier     |---> product[31:0]
  mode -------->|  (组合逻辑)       |         |
                +------------------+         |
                                             v
                +------------------+    +----------+
                |                  |<---| product  |
  acc_value --->|     Adder        |    +----------+
  (反馈)  ----->|  (组合逻辑)       |---> sum[31:0], carry
                +------------------+         |
                                             v
                +------------------+
  clk -------->|                  |
  rst_n ------>|  Accumulator     |---> acc_out[31:0] ---> out_31[31:0]
  en --------->|  (时序逻辑)       |---> carry_out     ---> carry
                |                  |---> out_ready      ---> out_ready
                +------------------+
```

### 2.2 模块划分

| 模块 | 文件 | 类型 | 功能 |
|------|------|------|------|
| `mac16` | `rtl/mac16.v` | 顶层模块 | 例化并连接各子模块 |
| `multiplier` | `rtl/multiplier.v` | 组合逻辑 | 16x16 乘法，支持有符号/无符号 |
| `adder` | `rtl/adder.v` | 组合逻辑 | 32-bit 加法，含溢出检测 |
| `accumulator` | `rtl/accumulator.v` | 时序逻辑 | 累加寄存器，异步复位 |

## 3. 接口说明

### 3.1 输入信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| `clk` | 1 | Input | 时钟信号 |
| `rst_n` | 1 | Input | 异步低有效复位，复位时输出及内部寄存器清零 |
| `en` | 1 | Input | 使能信号，高电平时执行乘累加运算 |
| `mode` | 1 | Input | 模式选择：0 = 无符号 MAC；1 = 有符号（补码）MAC |
| `a` | 16 | Input | 16-bit 输入操作数 A |
| `b` | 16 | Input | 16-bit 输入操作数 B |

### 3.2 输出信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| `out_31` | 32 | Output | 32-bit 累加输出结果 |
| `carry` | 1 | Output | 溢出标志位（无符号模式为进位，有符号模式为溢出） |
| `out_ready` | 1 | Output | 输出有效标志，表示当前周期有有效输出 |

## 4. 设计细节

### 4.1 乘法器 (multiplier)

- 将 16-bit 操作数扩展为 17-bit：无符号模式高位补 0，有符号模式符号扩展
- 利用 Verilog 的有符号乘法统一处理两种模式
- 取乘积低 32 位作为输出

### 4.2 加法器 (adder)

- 将乘积与当前累加值相加，产生 32-bit 和与溢出标志
- 无符号模式：检测 33 位结果的进位位
- 有符号模式：检测符号位溢出（两个同号数相加结果变号）

### 4.3 累加寄存器 (accumulator)

- 异步低有效复位：`rst_n` 为低时立即清零所有寄存器
- 使能控制：`en` 为高时更新累加值；为低时保持累加值不变，`out_ready` 拉低
- 寄存 carry 标志和 out_ready 信号

| 约束项 | 值 | 说明 |

| 最大转换时间 | 0.15 ns | |
| 异步复位 | false path | rst_n 不参与时序分析 |

## 8. 后端流程 (华大九天 EDA)

使用华大九天全流程工具的推荐步骤：

1. **仿真验证** (Empyrean ALPS/ALPS-GT) -- 功能仿真，确认 RTL 正确性
2. **逻辑综合** (Empyrean RC) -- 读入 RTL + SDC，映射到工艺库
3. **物理设计** (Empyrean Aether/Storm) -- 布局布线
4. **时序分析** (Empyrean TimingExplorer) -- 静态时序分析，确认满足 500MHz
5. **版图验证** (Empyrean Argus) -- DRC/LVS 检查
6. **导出 GDSII** -- 最终版图文件

## 9. 关键路径优化思路

- 乘法器和加法器均为组合逻辑，是关键路径所在
- 如时序不满足 500MHz，可考虑：
  - 插入流水线寄存器（乘法器输出打一拍），将组合路径拆分为两级
  - 使用 Booth 编码优化乘法器结构，减少部分积数量
  - 使用 Carry-Lookahead 或 Carry-Select 加速加法器
- 当前单周期设计结构简洁，适合面积优先的场景
